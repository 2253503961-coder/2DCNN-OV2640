# 实验二：手势数据标注

# 第一步：环境准备与硬件确认

在开始之前，我们需要确保电脑端的环境已经就绪：

## 安装依赖包

该脚本依赖 Python 的串口通信库。打开命令行终端，运行以下命令安装：

```bash
pip install pyserial
```

## 确认硬件状态

- 单片机（搭载 OV2640）已通过 USB 转串口模块连接到电脑。

- 单片机程序正在向串口源源不断地输出纯净的 JPEG 二进制流（就像您上一个 main\.c 里的操作）。

关键点：务必关闭电脑上其他的串口调试助手（如 XCOM、SSCOM），否则会由于端口占用导致脚本报错。

# 第二步：核心原理揭秘 —— 它是如何工作的？

面对串口传来的、没有固定长度的二进制“数据泥石流”，这个脚本使用了以下几个巧妙的机制来捕获图像：

1. **掐头去尾找图像**：脚本在 buffer 缓存中不断搜寻 JPEG 的标准头部标识符 FF D8 \(b\&\#39;\\xff\\xd8\&\#39;\) 和尾部标识符 FF D9 \(b\&\#39;\\xff\\xd9\&\#39;\)。只要找到这对标识符，就把中间的数据完整切片，这就相当于从泥石流中捞出了一张完整的照片。

2. **防死锁与丢包容错**：串口通信在极高波特率下难免丢包。如果丢了 FF D9，缓存会无限增长。脚本中 `if len\(buffer\) \&amp;gt; 102400:` 的设计非常精妙，一旦缓存超过约 100KB，就判断发生了严重的错位，直接清空重来，避免程序崩溃。

3. **物理目录即标签**：脚本利用了深度学习中最常用的“ImageFolder”结构。你要采集什么手势，它就自动建什么文件夹。文件夹的名字，就是未来训练模型时图片的标签（Label）。

# 第三步：具体操作

该脚本使用了 argparse 库，意味着你可以直接在命令行中通过附加参数来灵活控制采集过程。

## 基本语法格式

```bash
python collect_img.py --port <串口号> --baud <波特率> --gesture <手势名称> --count <数量>
```

## 场景 1：采集“石头 \(rock\)”手势

假设你的单片机插在 COM3 口，波特率 115200，你想采集 100 张石头的手势图：

```bash
python collect_img.py --port COM3 --baud 115200 --gesture rock --count 100
```

程序动作：它会自动在当前目录下创建一个 dataset/rock/ 文件夹，并开始抓拍。你只需要对着镜头不断变换手势角度。当抓满 100 张后，程序会自动停止。

## 场景 2：采集“剪刀 \(scissors\)”手势

保持硬件不动，立刻输入下一条命令采集剪刀，这次我们加大数量到 200 张：

```bash
python collect_img.py --port COM3 --gesture scissors --count 200
```

*注：如果不写 \-\-baud，脚本会自动使用默认值 115200*

# 第四步：输出目录解析 

当您完成 rock、paper、scissors 以及 none（无手势/背景）四个类的采集后，您的文件系统会自动变成这样：

```plaintext
📁 dataset/
  ├── 📁 rock/
  │    ├── rock_1680123456789.jpg
  │    ├── rock_1680123456890.jpg
  │    └── ...
  ├── 📁 paper/
  │    ├── paper_1680123500111.jpg
  │    └── ...
  ├── 📁 scissors/
  ├── 📁 none/
```

**如何采集高质量的 AI 数据集？**

- **多样性**：在执行脚本期间（比如等待采集那 100 张图时），不要让手僵住。要缓慢地转动手腕（多角度）、靠近和远离镜头（多尺度）、在不同位置移动（平移不变性）。

- **光照的变化**：在白天、晚上、强光、背光的环境下各跑几次脚本。对于 OV2640 这种宽容度较低的摄像头，光线对画质影响极大，模型需要见识过不同的光照。

- **千万别漏了 none 类**：很多人只采集手势，导致模型在没有人手的时候也会强行输出一个手势。务必使用 \-\-gesture none 采集大量的背景图（桌子、墙壁、空荡荡的房间），这能大幅降低模型的误报率。

# 实验三：手势识别模型训练与部署

# 一、前言

本讲义将详细讲解从深度学习环境配置、数据集准备，到模型训练、模型导出，最终完成单片机部署的全流程，重点适配轻量级模型在资源有限的单片机（SRAM仅几十KB）上的运行需求，步骤清晰可落地，适合嵌入式/AI初学者实操。

# 二、步骤拆解（全流程实操）

## 第一步：环境配置

在开始训练之前，需先在电脑上搭建完整的深度学习环境，核心依赖Python和PyTorch，具体操作如下：

1. 确认已安装Python（建议版本3\.7\-3\.10，兼容性最佳）；

2. 打开电脑终端（Windows：CMD/PowerShell；Mac/Linux：Terminal）；

3. 输入以下命令，安装PyTorch及视觉处理库（自动适配系统环境）：

```bash
pip install torch torchvision
```

提示：若安装速度较慢，可添加国内镜像源（如阿里云、清华源）加速安装。

## 第二步：准备数据集

本项目使用PyTorch的`datasets\.ImageFolder`方法读取数据，该方法会自动将文件夹名称作为分类标签，结合前期数据采集步骤，需严格按照以下结构准备数据集：

### 1\. 文件夹结构要求

确保当前代码运行目录下，存在名为`dataset`的文件夹，内部结构如下（建议用文件夹命名规范，避免中文）：

```plaintext
📁 dataset/
  ├── 📁 none/       (存放无手势的背景图，作为负样本)
  ├── 📁 paper/      (存放“布”手势图片，作为正样本1)
  ├── 📁 rock/       (存放“石头”手势图片，作为正样本2)
  └── 📁 scissors/   (存放“剪刀”手势图片，作为正样本3)
```

### 2\. 数据集注意事项

- 每类图片数量建议不少于几百张，数量越多，模型学习到的特征越全面，识别准确率越高；

- 图片采集需覆盖不同角度、不同光照条件，避免单一场景导致模型泛化能力差；

- 图片格式建议为JPG/PNG，无需提前预处理（后续代码会自动处理）。

## 第三步：理解模型架构（TinyVisionCNN）

为适配SRAM仅几十KB的单片机，本项目自定义了轻量级卷积神经网络（TinyVisionCNN），核心设计思路是“轻量化、低计算量”，具体架构拆解如下：

### 1\. 输入层（数据压缩）

将原始彩色大图，暴力压缩为 `1通道（灰度图）、28x28像素` 的小图，核心目的是减少计算量和内存占用，适配单片机有限的硬件资源。

### 2\. 特征提取层（核心部分）

采用2层卷积\+激活\+池化的结构，逐步提取手势特征，同时进一步压缩数据维度：

1. 第一层：3x3卷积（输出4通道）\+ ReLU激活函数 \+ 2x2最大池化；

2. 第二层：3x3卷积（输出8通道）\+ ReLU激活函数 \+ 2x2最大池化。

### 3\. 分类输出层

经过两次2x2最大池化后，特征图尺寸从28x28缩小为7x7；将7x7x8的特征图展平后，得到392个特征值（8\*7\*7）；最后通过全连接层（Linear），输出对应分类数量（本项目为4类：none、paper、rock、scissors）。

### 核心优势

模型参数极少、计算量低，无需复杂硬件支持，可直接适配各类资源有限的单片机（如STM32、51单片机等）。

## 第四步：开始训练

数据集和环境准备完成后，无需修改代码，直接在终端运行训练脚本，程序会自动完成全流程训练，具体操作如下：

### 1\. 运行训练命令

在终端切换到代码所在目录，输入以下命令，启动训练：

```bash
python train.py
```

### 2\. 自动执行流程（无需手动干预）

1. 数据预处理：将dataset文件夹下所有图片，自动转为单通道灰度图、缩放到28x28像素，并归一化到\[\-1, 1\]范围（适配模型输入要求）；

2. 数据划分：随机将数据集按8:2比例拆分，80%用于训练（模型学习特征），20%用于验证（测试模型学习效果）；

3. 迭代优化：使用Adam优化器（学习率0\.002），进行70轮（Epochs）训练，每轮训练会输出当前损失值（Loss）和准确率（Train Acc/Val Acc）。

### 3\. 训练输出示例

训练过程中，终端会实时输出以下内容，可通过输出判断训练效果：

```plaintext
Epoch [1/70] Loss: 1.3452 | Train Acc: 35.00% | Val Acc: 42.50%
Epoch [2/70] Loss: 1.1234 | Train Acc: 48.30% | Val Acc: 51.20%
...
Epoch [70/70] Loss: 0.1023 | Train Acc: 98.20% | Val Acc: 96.50%
```

### 4\. 训练异常排查

若训练过程中，**验证集准确率（Val Acc）迟迟上不去**，大概率是以下两种原因：

- 过拟合：训练集准确率很高，但验证集准确率低，可增加数据集数量、添加数据增强（如旋转、翻转）；

- 数据集不足/单一：采集的图片角度、光照、背景过于单一，模型无法学习到通用特征，需补充更多样化的图片。

## 第五步：模型导出与单片机部署（核心环节）

70轮训练结束后，脚本会自动进入“评估模式”，并生成两个核心文件，对应两种单片机部署路线，可根据自身需求选择，无需额外手动处理。

### 纯手写推理（灵活，无需依赖第三方框架）

#### 1\. 生成文件

脚本自动生成 `cnn\_vision\_weights\.h` 头文件，核心内容是将模型的权重（Weights）和偏置（Bias），从Python数组转为标准C语言`const float`数组。

#### 2\. 适用场景

适用于所有单片机（如STM32、51、ESP32等），无需依赖任何第三方AI框架，灵活性极高，适合嵌入式裸机开发。

#### 3\. 使用方法

1. 将 `cnn\_vision\_weights\.h` 头文件直接添加到单片机工程中；

2. 在C语言中，手写卷积计算、池化计算的for循环，或调用ARM CMSIS\-NN基础库；

3. 将头文件中的权重、偏置数组传入计算逻辑，即可完成手势识别推理，完全脱离庞大的AI框架依赖。

# 三、常见问题汇总

- 环境安装报错：若pip安装torch失败，可手动指定PyTorch版本，或检查Python版本是否兼容；

- 训练报错“找不到dataset文件夹”：检查文件夹路径是否正确，确保dataset文件夹与train\.py脚本在同一目录下；

- 部署后识别准确率低：检查数据集是否丰富，或模型训练轮数是否足够（可适当增加轮数至100轮）；

- 单片机内存不足：优先选择路线B，或进一步简化模型结构（减少卷积通道数）。

# 四、实操提示

1\. 建议先完成数据集准备，再进行环境配置和训练，避免因数据集问题导致训练白费；

2\. 训练过程中，可随时终止训练（Ctrl\+C），但需重新训练时，会覆盖之前的训练进度；

3\. 部署前，建议先在电脑上验证模型准确率（训练结束后的Val Acc），确保准确率达到90%以上再进行单片机部署。
